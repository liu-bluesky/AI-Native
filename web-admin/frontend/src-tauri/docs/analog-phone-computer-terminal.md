# 普通座机与电脑通信方案

目标：让一台普通座机不接运营商外线，只作为电脑旁边的语音终端使用。AI 工作完成后可以让座机响铃并播报结果；用户拿起听筒后可以通过按键或语音给电脑下达指令。

## 结论

采用 `普通座机 + FXS ATA 网关 + 本机电话服务 + AI Runner`。

不要使用 FXO。FXO 用来接真实电话线；本方案不需要拨打外部电话，只需要 FXS 给普通座机供电、振铃和传输音频。

```text
普通座机
  -> RJ11 电话线
FXS ATA 网关
  -> 局域网 / SIP
电脑本机电话服务
  -> AI 语音处理
  -> 本地 AI Runner
```

## 硬件

- 普通座机一台。
- FXS ATA 网关一台，例如 Grandstream HT801、HT802，或同类 SIP FXS 设备。
- 电脑与 FXS ATA 在同一个局域网内。

## 软件

- SIP/电话控制服务：可选 Asterisk、FreePBX，或后续自研轻量 SIP 服务。
- TTS：把 AI 完成情况转成语音，播放到座机听筒。
- STT：把用户语音转成文本指令。
- DTMF 按键解析：第一版建议优先支持按键，比语音识别更稳。
- 本地 AI Runner：接收指令、执行任务、回传结果。

## 工作流程

### AI 完成后通知

1. AI Runner 完成任务，生成简短摘要。
2. 电脑本机电话服务呼叫 FXS 分机。
3. FXS ATA 让座机响铃。
4. 用户拿起听筒。
5. TTS 播报完成情况。
6. 用户可按键选择下一步，例如重听、查看详情、继续执行或取消。

### 用户下达指令

1. 用户拿起座机听筒。
2. 电话服务检测摘机事件。
3. 系统播放菜单或等待语音。
4. 用户按键或说出指令。
5. 系统把 DTMF 或语音转写结果转换成结构化命令。
6. 本地 AI Runner 执行命令。
7. 执行结果通过听筒播报。

## 第一版建议

先做按键控制，不急着做完整语音识别。

```text
按 1：播报当前任务状态
按 2：继续执行当前任务
按 3：暂停当前任务
按 4：重听最近一次结果
按 9：确认高风险操作
按 0：取消并挂机
```

这样可以先验证座机、FXS、电脑、本地 Runner 的通信链路。链路稳定后，再增加语音识别。

## 安全边界

- 座机只能触发预定义指令，不能直接执行任意 shell 命令。
- 删除文件、部署、发消息、外部写入等高风险操作必须二次确认。
- 关键操作需要按 `9` 确认，或后续增加 PIN。
- 所有指令需要记录时间、来源、识别文本、执行结果和失败原因。
- 如果电脑端 Runner 离线，电话服务只能播报“当前电脑不可用”，不能伪造完成结果。

## 后续实现顺序

1. 配置 FXS ATA，把普通座机注册为本地 SIP 分机。
2. 电脑启动电话服务，能让座机响铃并播放固定音频。
3. 接入 AI Runner，AI 完成后触发响铃和播报摘要。
4. 支持 DTMF 按键菜单，完成状态查询、继续、暂停、取消。
5. 再接入 STT，让用户可以直接说自然语言指令。
