第 1 步
先在你本地准备生产配置文件。

在终端执行：

cd /Users/liulantian/self/ai-employee/docker
cp .env.prod.example .env.prod
chmod +x deploy.sh
ls -l .env.prod deploy.sh

第 2 步
先把本地的生产配置确认好。重点不是现在就连服务器，而是让后面打包和服务器启动时用同一套名字。

打开 .env.prod，至少确认这几项是这样：

API_IMAGE=ai_employee-api:latest
FRONTEND_IMAGE=ai_employee-frontend:latest
HOST_API_PORT=8000
HOST_FRONTEND_PORT=3000
DB_USER=admin
DB_NAME=ai_employee
DB_PORT=5432

DB_PASSWORD 你自己填，不要发给我。

改完后执行这条检查命令，把输出发我：

grep -E '^(API_IMAGE|FRONTEND_IMAGE|HOST_API_PORT|HOST_FRONTEND_PORT|DB_USER|DB_NAME|DB_PORT)=' .env.prod
第 3 步
现在开始打包后端镜像。这个过程会根据 Dockerfile.api:1 构建 ai_employee-api:latest。

在 docker/ 目录执行：

docker build -f Dockerfile.api -t ai_employee-api:latest ..

构建完成后，马上执行这条检查命令，把输出发我：

docker images | grep 'ai_employee-api'

docker build -f Dockerfile.frontend -t ai_employee-frontend:latest ..

• 第 4 步完成了，前端镜像也已经打出来了：

ai_employee-frontend:latest

第 5 步
现在把两个镜像导出成 tar 文件。这样后面才能上传到服务器再 docker load。

在 docker/ 目录执行：

docker save -o ../ai_employee-api_latest.tar ai_employee-api:latest
docker save -o ../ai_employee-frontend_latest.tar ai_employee-frontend:latest

导出完成后，执行这条检查命令，把输出发我：

ls -lh ../ai_employee-api_latest.tar ../ai_employee-frontend_latest.tar

第 6 步
开始备份数据。这里很重要，不只是数据库，还要备份技能卷和 API 数据卷。

先执行数据库备份：

./deploy.sh backup-db

执行完后，再执行这条检查命令，把输出发我：

ls -lh backup/ai_employee.sql
第 7 步
继续备份技能卷。这一步是为了保住上传的技能包文件，不然只恢复数据库，技能文件本体可能没了。

执行：

./deploy.sh backup-skill-volume

执行完后检查结果，把输出发我：

ls -la backup/mcp-skills-knowledge | head

你做完，我给你下一步。
第 8 步
现在备份 API 运行时数据卷。这一步通常会包含上传素材、导出产物、运行时文件等。

执行：

./deploy.sh backup-api-data-volume

执行完后检查结果，把输出发我：

ls -la backup/api-data | head

第 9 步
把备份目录打成一个压缩包，后面才能整体上传到服务器。

执行：

tar -czf docker-backup-$(date +%Y%m%d-%H%M%S).tar.gz backup

执行完后，再检查一下当前生成了什么，把输出发我：

ls -lh docker-backup-\*.tar.gz

你做完，我给你下一步。

第 10 步
先确认你能正常登录服务器。执行：

第 11 步
先不要在服务器里操作。你先回到你本地 Mac 的终端窗口，在本地把文件上传到服务器。

在你本地终端执行这 3 条命令：

scp /Users/liulantian/self/ai-employee/ai_employee-api_latest.tar root@64.81.113.174:/root/
scp /Users/liulantian/self/ai-employee/ai_employee-frontend_latest.tar root@64.81.113.174:/root/
scp /Users/liulantian/self/ai-employee/docker/docker-backup-20260330-170927.tar.gz root@64.81.113.174:/root/

然后再把整个 docker/ 目录传上去：

scp -r /Users/liulantian/self/ai-employee/docker root@64.81.113.174:/root/ai-employee/

传完后，重新回到服务器终端，执行：

ls -lh /root
ls -lh /root/ai-employee
ls -lh /root/ai-employee/docker | head

第 12 步
先把服务器上的目标目录创建出来，然后把本地 docker/ 目录上传上去。

先在服务器终端执行：

mkdir -p /root/ai-employee

然后切回你本地 Mac 终端，执行：

scp -r /Users/liulantian/self/ai-employee/docker root@64.81.113.174:/root/ai-employee/

上传完成后，回到服务器终端执行：

ls -lh /root/ai-employee
ls -lh /root/ai-employee/docker | head

第 13 步
进入服务器上的 docker/ 目录，准备生产配置文件。

在服务器终端执行：

cd /root/ai-employee/docker
cp .env.prod.example .env.prod
chmod +x deploy.sh
ls -l .env.prod deploy.sh

现在先不要急着启动。先把服务器上的两个镜像导入 Docker。

导入完成后，执行这条检查命令，把输出发给我：

你做完，我给你下一步。
