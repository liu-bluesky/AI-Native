# 后端智能体迁移与删除记录

## 目标

后端 `web-admin/api/services/agent_runtime/` 已于 2026-07-16 删除。涉及模型循环、工具调用、权限确认、运行事件、暂停恢复和工作区操作的能力统一由 Tauri `liuagent_core` 承担。

旧 `.ai-employee/agent-runtime-v2/` 历史数据继续保留，不随源码删除。

## 当前执行边界

| 功能点 | 迁移后的执行位置 | 后端职责 |
| --- | --- | --- |
| 项目聊天 | Tauri `liuagent_core` | 项目、模型和上下文数据接口 |
| 无项目通用对话 | Tauri `liuagent_core` | 模型供应商桌面配置接口 |
| 全局助手文本对话 | Tauri `liuagent_core` | 系统配置、历史和语音服务 |
| 项目工作日志 AI 总结 | Tauri `liuagent_core` | 工作日志数据读取与保存 |
| 服务端动态定时任务决策 | 后端一次性 LLM JSON 调用 | 只生成受限动作计划，不进入工具循环、不写聊天历史 |
| 文件、命令、网络和 MCP 工具 | Tauri `liuagent_core/tools` | 不执行用户本机工具 |
| 工具权限确认 | Desktop Permission Gate | 不保存新桌面任务的 Runtime 权限 |
| 工作区永久授权 | 桌面端按工作区保存 | 不调用后端 workspace-trust API |
| 暂停与继续 | Desktop checkpoint/state/outbox | 仅接收同步后的结构化摘要 |

## 第二阶段隔离措施

- 桌面 Runtime 的状态、权限、离线缓存、Web 工具配置和机器人配置统一迁移到 `.ai-employee/desktop-agent-runtime/`。
- 首次访问时只从旧 `.ai-employee/agent-runtime-v2/` 复制桌面自有目录，并写入版本标记；后续不重复迁移、不覆盖新目录数据。
- 旧目录中的后端 `task-runs`、`events`、`transcripts` 不会复制到桌面命名空间；`permissions/` 仅迁移根目录桌面会话文件，明确排除后端 `permissions/decisions/` 子目录。
- `.ai-employee/query-mcp/active-sessions`、`session-history` 和 `outbox` 继续使用既有 canonical 路径，不随 Runtime 命名空间迁移。
- 后端四个旧编排入口固定拒绝执行，不再提供 Runtime 回滚开关。
- 全局助手文本链已接入桌面 Runtime 事件、暂停、恢复快照和本机工具授权续跑；后端 WebSocket 仅保留语音转写与浏览器桥接兼容。

## 删除后兼容边界

- 已存在的后端 Runtime 历史消息、权限卡片和恢复事件仍可展示。
- 后端 Runtime API 返回 `410 Gone`，不再执行查询、恢复或权限写入。
- 前端兼容处理旧 `agent_runtime_*` 事件，但新桌面任务不得产生这些事件。

## 自动检查

运行：

```bash
npm run test:desktop-agent-migration
```

检查内容包括：

1. 项目聊天的新请求固定进入桌面 Runtime。
2. 无项目通用对话不再调用 `/projects/chat/global` 生成文本。
3. 全局助手文本对话使用 `startNativeLiuAgentLocalChat`。
4. 项目工作日志总结使用桌面 Runtime。
5. 工作区信任不再调用后端 Agent Runtime API。
6. 后端 `agent_runtime` 源码目录已经删除。
7. 桌面 Runtime 使用独立存储命名空间，且迁移不复制后端运行记录。
8. 后端四个智能体编排入口固定拒绝新运行，且不存在回滚开关。

## 2026-07-15 真实桌面观察记录

- 当前项目旧 `.ai-employee/agent-runtime-v2/` 共 `277M`，迁移后 `.ai-employee/desktop-agent-runtime/` 为 `1.9M`，旧目录保持不变。
- 旧 `permissions/decisions/` 共 `271M`、318 个后端权限历史文件；真实迁移后新目录不存在 `permissions/decisions/`。
- 新目录不存在 `task-runs`、`events`、`transcripts`，只包含项目缓存、模型配置、会话缓存、桌面 sessions 和迁移版本标记。
- HOME 下机器人连接器、监听上下文和 Web 工具配置的新旧文件校验和一致，说明一次性迁移完整。
- `npm run tauri:dev` 可启动真实“AI 员工工厂”窗口，进程持续运行且无启动崩溃。
- 桌面启动后 `$HOME/.ai-employee/desktop-agent-runtime/bots/feishu-listener.jsonl` 持续更新，旧 `$HOME/.ai-employee/agent-runtime-v2/bots/feishu-listener.jsonl` 未再更新。
- 后端 Runtime 配置开关、启动恢复和 Python builtin tools fallback 已移除。

## 删除门槛完成情况

2026-07-16 已完成并确认：

1. 桌面端项目聊天、全局助手和工作日志连续运行稳定。
2. 权限允许一次、会话授权、工作区信任和拒绝流程均通过验证。
3. 暂停、应用重启、继续执行和后台任务恢复均通过验证。
4. 后端停止或不可访问时，本地模型与本地工具任务仍可继续；依赖后端数据的操作必须明确提示。
5. 仓库搜索确认没有新产品入口创建 `AgentTaskRuntime`。
6. 后端 Runtime 历史数据完成归档或确认无需迁移。
7. 删除范围、影响、备份和恢复方式再次获得用户明确确认。

## 删除结果

已清理 Python 生产导入、启动恢复、Runtime 配置开关、Python builtin tools 和专用测试。旧 Runtime API 保留路由外形但固定返回 `410 Gone`，用于让历史前端得到明确退役响应。服务端动态定时任务保留独立的一次性 LLM JSON 决策，不具备工具调用、暂停恢复或 Runtime 状态；所有 Tauri `liuagent_core` 能力和历史数据均保留。
