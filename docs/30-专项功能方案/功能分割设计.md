# 功能分割设计补充

> 解决员工创建与人设管理功能重合问题，增加 MCP 服务状态查看功能。

---

## 1. 员工创建 vs 人设管理 分割方案

### 1.1 当前问题

```
当前实现（功能重合）：

员工创建时：
├── name, description
├── skills[]
├── rules{}
└── persona:                        ← 人设字段直接嵌入
    ├── persona_id                  ← 引用人设
    ├── tone                        ← 又直接设置
    ├── verbosity                   ← 又直接设置
    ├── language                    ← 又直接设置
    ├── style_hints                 ← 又直接设置
    ├── decision_policy             ← 又直接设置
    └── drift_control               ← 又直接设置

人设管理（独立）：
├── 创建人设模板
├── tone, verbosity, language...
└── decision_policy, drift_control...

问题：两处都可以设置人设，功能重合，用户困惑
```

### 1.2 分割方案

**方案：员工只引用人设，人设管理作为模板库**

```
┌─────────────────────────────────────────────────────────────┐
│ 员工创建                                                      │
│                                                              │
│ 必填字段：                                                    │
│ ├── name: string                                             │
│ ├── description: string                                      │
│ ├── skills[]: 引用技能 ID 列表                                │
│ ├── persona_id: string  ← 必须引用预设人设                    │
│ └── rules{}: 规则配置                                         │
│                                                              │
│ 可选字段：                                                    │
│ ├── memory: 记忆配置                                          │
│ └── evolution: 进化配置                                       │
│                                                              │
│ ❌ 不再包含：tone, verbosity, style_hints 等人设字段          │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 人设管理（模板库）                                             │
│                                                              │
│ 人设模板：                                                    │
│ ├── id: string                                               │
│ ├── name: string                                             │
│ ├── tone: professional | friendly | strict | mentor          │
│ ├── verbosity: verbose | concise | minimal                   │
│ ├── language: string                                         │
│ ├── style_hints[]: string[]                                  │
│ ├── behaviors[]: string[]                                    │
│ ├── decision_policy: 决策策略                                 │
│ └── drift_control: 偏移控制                                   │
│                                                              │
│ 预设模板：                                                    │
│ ├── persona-frontend-expert: 前端专家                        │
│ ├── persona-backend-expert: 后端专家                         │
│ ├── persona-test-engineer: 测试工程师                        │
│ ├── persona-devops: 运维专家                                 │
│ └── persona-mentor: 技术导师                                 │
└─────────────────────────────────────────────────────────────┘
```
