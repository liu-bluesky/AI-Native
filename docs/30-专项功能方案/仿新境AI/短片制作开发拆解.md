# 短片制作开发拆解

## 1. 目标

- 项目 ID：`proj-d16591a6`
- 目标页面：`/materials/studio`
- 当前承载文件：`web-admin/frontend/src/views/projects/ProjectShortFilmStudio.vue`
- 目标：把“仿新境AI”原型中的短片创作主流程，收敛成适配当前项目工作区风格的可实施版本。

本拆解不覆盖：

- 首页营销页
- VIP 会员模块
- 平台级商业化转化

这些保留在外围能力，不进入本轮 `/materials/studio` 主工作区。

## 3. 实施范围

本轮建议只做短片制作的工作流主线：

1. 剧本输入
2. 智能分章
3. 美术设定
4. 元素提取与角色配置

## 4. 页面拆分建议

建议把 `/materials/studio` 页面拆成 4 个主区块，而不是直接复刻原型的重后台布局。

### 4.1 Studio 顶部上下文条

作用：

- 展示当前项目名称
- 展示当前阶段
- 提供少量全局动作

建议内容：

- 标题：`短片制作`
- 副标题：当前阶段说明
- 阶段概览：`剧本 / 设定 / 分镜 / 导出`
- 次动作：
  - 返回素材库
  - 打开作品页

不建议放：

- 大英雄区
- 大面积说明文案
- 重统计卡

### 4.2 主流程步骤条

步骤条固定为 4 步：

1. 剧本创作
2. 美术设定
3. 分镜制作
4. 导出视频

实现要求：

- 支持点击切换已完成步骤
- 未满足前置条件时阻止跳转并给出提示
- 当前步骤状态至少有：
  - `pending`
  - `active`
  - `completed`
  - `blocked`

### 4.3 当前步骤工作区

核心内容区域只展示当前步骤，不同时堆叠多个步骤面板。

建议结构：

- 左侧：当前步骤主操作区
- 右侧：当前步骤辅助区 / 状态摘要区

其中：

- 步骤一右侧放章节列表
- 步骤二右侧放设定进度
- 步骤三右侧放生成历史
- 步骤四右侧放导出信息

### 4.4 底部动作区

统一动作：

- `上一步`
- `下一步`
- 当前步骤主操作按钮

要求：

- 主按钮始终只保留一个
- 次要动作回退为次按钮或文本按钮

## 5. 功能模块拆解

### 5.1 模块 A：剧本输入与分章

页面目标：

- 收集剧本内容
- 生成章节
- 确认后进入设定阶段

前端模块建议：

- `StudioScriptEditor`
- `StudioAspectRatioSelector`
- `StudioChapterList`
- `StudioChapterPreviewDialog`

最小功能：

- 文本输入
- 文件导入：`txt/md`
- 字数统计
- 比例选择
- 智能分章
- 章节预览

数据结构建议：

```ts
type StudioScriptDraft = {
  content: string
  sourceFileName: string
  aspectRatio: "16:9" | "9:16" | "1:1" | "3:2" | "2:3"
}

type StudioChapter = {
  id: string
  index: number
  title: string
  summary: string
  content: string
}
```

完成条件：

- 有剧本内容
- 至少生成 1 个章节

### 5.2 模块 B：美术设定与元素提取

页面目标：

- 选择视觉风格
- 生成角色/场景/道具
- 支持角色级配置

前端模块建议：

- `StudioStyleSelector`
- `StudioElementTabs`
- `StudioExtractionPanel`
- `StudioExtractionResultDialog`
- `StudioCharacterConfigDialog`

最小功能：

- 风格多选
- 元素类型切换：角色 / 场景 / 道具
- 提取参数配置：模型 / 时长 / 画质
- AI 提取
- 手动新增元素
- 角色详情四视图
- 角色级音色配置

数据结构建议：

```ts
type StudioStyleOption = {
  id: string
  label: string
  selected: boolean
}

type StudioElementKind = "role" | "scene" | "prop"

type StudioElement = {
  id: string
  kind: StudioElementKind
  name: string
  status: "detected" | "pending" | "manual"
  previewUrl?: string
  metadata?: Record<string, unknown>
}

type StudioCharacterVoicePreset = {
  value: string
  label: string
  hint: string
}
```

完成条件：

- 至少完成一次元素提取或手动补录
- 角色类元素可保存配置

### 5.3 模块 C：分镜生成与配音

页面目标：

- 以章节为单位生成分镜
- 对分镜补齐配音
- 选定进入导出的镜头集合

前端模块建议：

- `StudioChapterStoryboardPicker`
- `StudioStoryboardGrid`
- `StudioVoiceGenerateDialog`
- `StudioGenerationHistoryPanel`

最小功能：

- 章节切换
- 分镜参数配置：模型 / 时长 / 画质
- 背景音乐 / 音效开关
- 分镜生成
- 分镜重生成
- 分镜配音
- 分镜勾选
- 已选数量统计

关键规则：

- 未配音分镜不可选入导出集合
- 配音成功后允许勾选

数据结构建议：

```ts
type StudioStoryboard = {
  id: string
  chapterId: string
  title: string
  previewUrl?: string
  durationSeconds: number
  hasVoice: boolean
  selected: boolean
  status: "draft" | "ready" | "failed"
}

type StudioVoiceConfig = {
  model: string
  timbre: string
  speed: string
}
```

完成条件：

- 至少有 1 个已配音且被选中的分镜

### 5.4 模块 D：导出前编排

页面目标：

- 编排已选分镜
- 调整时长
- 预览合成结果
- 提交导出任务

前端模块建议：

- `StudioTimelineEditor`
- `StudioPreviewPlayer`
- `StudioExportPanel`
- `StudioAddToTimelineDialog`

最小功能：

- 已选分镜列表
- 时长编辑
- 视频轨道
- 音频轨道
- 历史分镜回添时间线
- 导出信息展示
- 导出参数选择

导出参数建议：

- 编码：`MP4 (H.264)`、`MP4 (H.265)`
- 清晰度：`720p`、`1080p`、`4K`

数据结构建议：

```ts
type StudioTimelineClip = {
  id: string
  storyboardId: string
  title: string
  startSeconds: number
  durationSeconds: number
  track: "video" | "audio"
}

type StudioExportConfig = {
  format: "mp4-h264" | "mp4-h265"
  resolution: "720p" | "1080p" | "4k"
}
```

完成条件：

- 至少存在 1 个视频轨道片段
- 已选择导出参数

#### 5.4.1 截至 2026-03-21 的已落地能力

当前导出前编排模块已实际具备以下能力：

- `StudioMergedPreview` 已落地为可交互时间线预览，而非静态展示。
- 视频轨道片段支持选中、拖拽排序、右侧 resize handle 调整时长、右键删除。
- 预览区已接入播放器控制：上一段、播放/暂停、下一段、回到开头。
- 时间线播放头已改为单根连续覆盖线，支持拖拽，ruler 顶部时间气泡也支持拖拽定位。
- 音频轨已拆分为 `旁白` 和 `背景音乐` 两行展示。
- 背景音乐已在预览阶段实际播放，不再只是轨道展示。
- 旁白已按当前时间线命中的 segment 动态切换音源并参与预览播放。
- 视频 / 旁白 / 背景音乐 三路都已支持单独静音和音量控制。
- 音量控制已压缩到左侧轨道标签区域，喇叭按钮常驻，滑杆通过悬浮面板展开。

#### 5.4.2 本轮已解决的导出阶段交互问题

本轮围绕时间线预览与导出编排，已修复以下高频问题：

- 视频片段过窄时内容文字被遮挡，现已改为保留标题优先展示，并移除片段内时间文本。
- 窄片段使用浮动标题标签，避免文字被裁切到不可读。
- 播放主按钮 hover 后文字看不清的问题已修复。
- 播放头原先按视频 / 旁白 / 背景音乐分段显示，现已统一为连续直线。
- 时间线刻度线被片段表面和激活态高亮压住的问题已修复，刻度线层级已抬到片段之上。
- ruler 刻度拖拽与主体播放头拖拽都已接入统一时间同步。
- 片段拖拽调时长时“不能及时跟手”的问题已修复，交互更新更连贯。
- 两个视频或旁白文件在边界切换时，旧媒体 `pause` 事件导致来回暂停播放的死循环问题已加保护处理。
- 左侧轨道标签区的喇叭按钮、文字和音量面板已重新整理，避免不对齐和鼠标一离开就收起。

#### 5.4.3 当前仍需继续观察的点

- 轨道标签内的音量控制虽然已压缩，但仍需要继续观察桌面端与窄屏下的误触和可达性。
- 预览播放链路目前已覆盖视频、旁白、背景音乐三路，但复杂多段旁白连续切换仍需要更多真实素材回归。
- 正式导出链路与预览链路已经打通到同一时间线语义，但“完整真实素材合成”仍属于后续阶段任务。

## 10. 风险与待定项

当前有 4 个必须先明确的产品问题：

1. “导出视频”是否保留为步骤四页内能力，还是拆成独立导出页。
2. 元素提取后，角色/场景/道具是否都需要进入素材库。
3. 分镜生成产物是图片、视频，还是两者并存。
4. 角色级音色配置与分镜级配音参数，最终谁优先。

如果这 4 个问题不收口，前端实现很容易返工。

## 11. 验收标准

完成 `/materials/studio` 首版后，至少满足：

- 用户可在一个页面内完成 4 步流程切换
- 每一步都有明确主操作
- 未满足前置条件时，下一步不可误入
- 分镜未配音时不能进入导出集合
- 导出区可展示时间线、导出参数、预计消耗
- 页面风格与 `/materials` 工作区一致，不退回重后台原型风格

## 12. 结论

当前最合理的实现方式，不是“直接照抄 HTML 原型”，而是基于现有 `/materials` 工作区做模块化重建：

- 用 `ProjectShortFilmStudio.vue` 承接 4 步主流程
- 用组件化方式拆分剧本、设定、分镜、导出
- 用统一 studio 状态驱动步骤推进
- 用 `/materials` 和 `/materials/works` 承接沉淀与结果

这样既能保留原型的业务主线，也能和当前项目已经形成的低噪音工作区风格保持一致。

## 13. 下次开发计划（周一续做）

下次开发不再从“搭结构”开始，而是继续围绕导出阶段时间线和预览播放器做稳定性补完，按下面顺序推进：

### 13.1 优先处理的回归与体验问题

- 继续回归时间线左侧轨道标签区，确认喇叭按钮、文字、滑杆在桌面端和窄宽度下都不遮挡、不跳位。
- 继续验证片段激活态、hover 态、拖拽态和 resize 态的层级关系，避免再次压住刻度线、播放头或浮动标题。
- 继续验证视频 / 旁白 / 背景音乐在片段边界切换时的事件顺序，重点看连续播放、手动拖拽、回到开头、下一段切换这几条链路。
- 继续验证音量控制在 hover、点击静音、解除静音、拖动滑杆后的状态同步，避免 UI 状态和真实媒体元素脱节。

### 13.2 预览播放器链路继续补齐

- 给旁白多片段连续切换补更多保护，避免真实素材下出现短暂断音、重复切换或错误回退。
- 检查视频静音和暂停链路是否存在互相干扰，避免只想改音量却把播放器状态打断。
- 评估是否需要把轨道音量面板从纯 hover 再升级为“点击展开 + 外部收起”模式，提高可操作稳定性。
- 如果旁白轨开始有真实数据，补一轮针对多段 voice 的真实页面回归。

### 13.3 时间线交互继续完善

- 继续优化 resize 跟手性，必要时引入更平滑的拖拽预览宽度，而不是完全依赖提交后的时长回写。
- 继续检查 ruler 和主体播放头拖拽在不同轨道高度下的命中范围，避免拖拽时跳点或误触。
- 评估是否需要进一步细化刻度与网格样式，让激活片段、播放头、刻度线三者同时可见且不互相抢焦点。

- 把本轮时间线预览修复整理成一份更明确的回归测试清单，覆盖播放、拖拽、音量、边界切换、导出前检查。
