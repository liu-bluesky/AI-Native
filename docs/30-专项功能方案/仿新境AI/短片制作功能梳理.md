# 短片制作功能梳理

## 1. 文档说明

- 项目 ID：`proj-d16591a6`
- 解析来源：
  - `docs/30-专项功能方案/仿新境AI/index (1).html`
  - `docs/30-专项功能方案/仿新境AI/prd.md`
- 目标：基于原型 HTML 还原“短片制作”相关的页面结构、主流程、功能模块、核心状态和落地边界，供后续在当前项目中实现。

## 2. 原型整体信息架构

HTML 原型不是单页“短片制作页”，而是一个小型短片创作平台，包含 4 类页面：

| 页面 | 入口 | 作用 |
| --- | --- | --- |
| 首页 `page-home` | 侧边栏首页 / Hero CTA | 平台介绍、能力说明、热门作品、转化入口 |
| 新建短片 `page-workshop` | 侧边栏“新建短片” | 短片从剧本到导出的主创作流程 |
| 分镜编辑与导出 `page-export` | 原型单独页面 | 对已选分镜做二次时长编辑与合并导出 |
| 我的作品 `page-works` | 侧边栏“我的作品” | 已创建短片项目的列表管理 |

额外还有两个外围模块：

- 顶部“功能导航”下拉，提供 `新建短片`、`我的作品` 快捷入口
- VIP 会员弹窗，用于充值、支付和权益转化

此外原型还有一层平台级公共壳层能力：

- 语言切换
- 积分余额展示
- 版本标识
- 当前登录用户头像
- AI 服务状态与“今日已生成分镜数”运行状态卡

## 3. 短片制作主流程

原型把短片创作拆成 4 个顺序步骤：

1. 剧本创作
2. 美术设定
3. 分镜制作
4. 导出视频

这 4 步都挂在 `page-workshop` 内，用步骤条切换；其中“导出视频”又额外提供了 `page-export` 的单独导出编辑页。

### 3.1 步骤一：剧本创作

目标：完成原始剧本输入和章节拆分。

功能点：

| 功能 | 说明 |
| --- | --- |
| 剧本输入 | 大文本框输入剧本，提示最大 25000 字 |
| 文件导入 | 支持 `txt`、`md` 导入 |
| 画面比例选择 | `16:9`、`9:16`、`1:1`、`3:2`、`2:3` |
| 智能分章 | 点击后把剧本拆成章节列表 |
| 章节列表 | 展示章节标题和摘要 |
| 章节预览 | 点击章节打开预览弹窗 |
| 下一步 | 进入美术设定 |

实现观察：

- `autoSplitChapters()` 目前是前端 mock，直接生成 3 个章节样例。
- 文件导入用 `FileReader` 直接读文本。
- 章节预览走弹窗，不是页内详情。

### 3.2 步骤二：美术设定

目标：基于剧本完成风格选择和元素提取，为后续分镜生成准备设定素材。

功能点：

| 功能 | 说明 |
| --- | --- |
| 风格多选 | 水墨、3D 国潮、电影超写实、卡通、赛博朋克、油画、水彩、像素风 |
| 元素分类切换 | `角色`、`场景`、`道具` 三类 Tab |
| 提取参数配置 | 模型、时长、画质 |
| 元素提取 | 点击“开始提取”后打开提取结果弹窗 |
| 元素展示 | 页内展示角色卡片、场景卡片、道具卡片 |
| 手动新增元素 | 三类元素区都预留 `+` 卡片，用于手动补录 |
| 角色详情 | 角色支持打开“四视图”详情弹窗 |
| 角色级音色配置 | 在角色详情里为角色选择音色预设 |
| 角色配置保存 | 在角色详情里保存配置 |
| 进度面板 | 显示风格选择、元素提取、角色制作进度 |

实现观察：

- 提取结果同样是前端 mock，结果里只有 `已识别/待确认` 状态。
- 角色详情弹窗强调“四视图”，说明角色资产不是纯文本标签，而是需要结构化设定。
- 三类元素区都提供虚线 `+` 卡片，说明原型不仅支持 AI 提取，也预留了人工补建入口。
- 角色详情里的“音色配置”说明角色和配音关系不是临时绑定，而是允许先沉淀角色级声音设定。
- 当前角色音色预设至少包含：`温柔女声`、`成熟男声`、`清脆童声`，并带有角色适配说明。
- 这一阶段本质上在做“全局视觉设定”和“素材基底生成”。

### 3.3 步骤三：分镜制作

目标：按章节生成分镜，并补齐配音，筛选进入合成阶段的镜头。

功能点：

| 功能 | 说明 |
| --- | --- |
| 章节选择 | 先选章节，再展示对应分镜 |
| 音效/配乐开关 | 背景音乐、音效开关 |
| 分镜生成参数 | 模型、时长、画质 |
| 分镜展示区 | 网格展示分镜卡片 |
| 配音生成 | 每个分镜可单独生成配音 |
| 分镜重生成 | 提供重新生成动作 |
| 分镜勾选 | 只有“已配音”分镜才允许勾选 |
| 已选计数 | 统计已选分镜数量 |
| 生成历史 | 右侧显示历史分镜记录 |

关键业务规则：

- 分镜必须先生成配音，才能进入“已选”状态。
- 分镜筛选是后续导出前的前置门槛，不是所有生成结果都会进入成片。

实现观察：

- `storyboards` 数据只包含 `id/title/hasVoice/selected`，说明原型阶段还没细化 prompt、图片、视频、镜头脚本等结构。
- `regenerateStoryboard()` 目前只有提示，没有真实重生成功流。

### 3.4 步骤四：导出视频

目标：对已选分镜做最终编排、预览和导出。

功能点：

| 功能 | 说明 |
| --- | --- |
| 分镜勾选列表 | 左侧列出已入选分镜 |
| 时长编辑 | 每个分镜可单独改秒数 |
| 总时长汇总 | 汇总总时长和已选数量 |
| 合并视频预览 | 中间区域展示预览播放器 |
| 视频轨道 | 支持按时间轴查看分镜片段 |
| 音频轨道 | 展示背景音乐轨道 |
| 历史分镜库 | 右侧可从历史分镜回添到时间线 |
| 添加到时间线 | 点击历史分镜后弹窗确认 |
| 导出信息 | 总时长、分镜数量、预计积分消耗 |
| 导出参数 | 编码格式、清晰度 |
| 导出视频 | 最终导出动作 |

实现观察：

- 时间轴支持 `draggable` 和左右 resize handle，说明意图是做可编排时间线，而不只是表单式时长配置。
- “历史分镜添加到视频轨道”是导出阶段的核心补充能力，说明分镜库与时间线不是一对一关系。
- 导出参数在原型里已经给出具体枚举：
  - 编码格式：`MP4 (H.264)`、`MP4 (H.265)`
  - 清晰度：`720p`、`1080p`、`4K`
- 导出信息不仅展示总时长和分镜数量，还包含 `预计消耗积分`。
- `page-export` 里又单独做了一版“分镜编辑与导出”，说明产品还在“导出功能放在主流程第四步”还是“拆成独立编辑页”之间摇摆，这是后续实现前必须收口的产品决策点。

## 4. 辅助与外围模块

### 4.1 首页转化模块

首页不是业务工作台，而是转化页，主要承担：

- Hero 区引导“立即创作”
- 核心能力说明：剧本创作、美术设计、分镜设计、短片合成
- 热门作品展示
- 平台卖点说明：AI 智能生成、多风格支持、智能配音、一键导出

结论：

- 这部分更偏营销和平台介绍，不是当前项目 `/materials/studio` 的核心工作区内容。

### 4.2 我的作品

“我的作品”是成片结果管理页，功能包括：

- 搜索作品
- 类型筛选
- 排序
- 作品卡片展示
- 状态标签，如“已完成”“制作中”

结论：

- 它承接的是“项目/成片结果管理”，而不是创作过程本身。

### 4.3 VIP 会员模块

外围还有完整的会员转化链路：

- 会员入口卡
- 套餐选择
- 支付方式选择
- 支付确认

当前原型里给出的具体枚举包括：

- 套餐：`周卡`、`月卡`、`年卡`、`终身卡`
- 支付方式：`微信支付`、`支付宝`、`QQ支付`、`银行卡`

结论：

- 这是商业化模块，和短片制作主流程解耦。
- 当前项目如果只做创作能力落地，可以先不纳入 `/materials` 主流程。

## 5. 原型里的核心对象与状态

从 HTML 与脚本可以抽出这些关键业务对象：

| 对象 | 当前原型字段/状态 | 说明 |
| --- | --- | --- |
| 剧本 | 内容、文件导入、画面比例 | 创作入口 |
| 章节 | `id/title/content` | 剧本拆分结果 |
| 美术风格 | 多选风格 | 全局视觉设定 |
| 元素 | 角色/场景/道具 | 从剧本里抽取出的素材对象 |
| 角色配置 | 四视图、音色/角色属性 | 偏资产化设定 |
| 分镜 | `id/title/hasVoice/selected` | 镜头级创作单元 |
| 配音 | 模型、音色、语速 | 分镜进入导出前的必要条件 |
| 时间线片段 | 时长、位置、可拖拽 | 成片编排单元 |
| 导出任务 | 格式、清晰度、积分消耗 | 最终交付单元 |
| 作品 | 标题、更新时间、状态 | 成果管理单元 |

最重要的状态约束有两条：

1. 剧本 -> 章节 -> 美术设定 -> 分镜 -> 导出，是明确的阶段推进链路。
2. 分镜只有在“已配音”后才能被勾选进入导出集合。

## 6. 原型中的伪实现与未落地部分

当前 HTML 主要是交互原型，不是可直接接入后端的真实实现，证据包括：

- `chapters`、`storyboards` 都是前端写死数组
- 大量动作是 `alert()`，例如：
  - 元素提取成功
  - 配音生成成功
  - 分镜重新生成
  - 添加到时间线
- 页面切换靠 `switchPage()` 和 DOM 显隐，不是真实路由
- `nextStep()` / `prevStep()` 只切步骤，没有完整校验链
- 导出页存在重复表达：既有 `workshop` 第四步，也有独立 `page-export`

因此后续实现时不能把原型里的交互直接当产品事实，需要做一轮产品定稿。

## 7. 对当前项目的落地映射建议

结合当前仓库已有路由，原型功能更适合这样映射：

| 原型能力 | 当前项目建议承接位置 |
| --- | --- |
| 新建短片主流程 | `/materials/studio` |
| 作品列表 | `/materials/works` |
| 历史素材/图像/分镜沉淀 | `/materials` |
| 首页营销页 | 不建议放进 `/materials` 工作区 |
| VIP 会员 | 先不进入本轮实现 |

进一步建议：

### 7.1 `/materials/studio` 应承接的主流程

- 剧本输入与导入
- 智能分章
- 美术设定
- 元素提取
- 分镜生成
- 配音生成
- 导出前编排

### 7.2 `/materials` 应承接的内容

- 正式素材沉淀
- 分镜/图片/视频资产回看
- 来源回跳
- 资产筛选与编辑

### 7.3 `/materials/works` 应承接的内容

- 成片结果列表
- 项目级作品状态
- 导出结果回看

## 8. 结论

这个原型的核心不是“单一短片页面”，而是一个完整的 AI 短片创作工作流，真正有实现价值的主线是：

`剧本输入 -> 智能分章 -> 美术设定 -> 元素提取 -> 分镜生成 -> 配音补齐 -> 时间线编排 -> 导出成片 -> 作品管理`

对当前项目来说，最合理的拆法不是把整份原型完整照搬，而是：

- 把“短片创作主流程”收敛进 `/materials/studio`
- 把“作品管理”放进 `/materials/works`
- 把“营销首页、热门作品、VIP”视为外围能力，暂不混入创作工作区

这样才能和当前项目已经建立的 `素材库 / 短片制作 / 我的作品` 路由结构对齐。
