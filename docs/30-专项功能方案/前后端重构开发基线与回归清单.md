# 前后端重构开发基线与回归清单

> 日期：2026-04-25  
> 关联计划：`docs/30-专项功能方案/前后端对标Claude重构计划.md`  
> 本轮原则：优先低风险拆分，保持行为不变，避免覆盖工作区已有未提交改动。

## 1. 当前基线

- 前端主风险文件：`web-admin/frontend/src/views/projects/ProjectChat.vue`，承担聊天页、设置中心、任务树、终端镜像、MCP 接入、代码预览等职责。
- 后端主风险文件：`web-admin/api/routers/projects.py`，承担项目、聊天、素材、短片工作室、导出与任务树相关路由。
- 运行时基础已存在：`web-admin/api/services/runtime/`，后续应作为 Runtime Core 收口位置。
- 任务树反馈基础已存在：`web-admin/frontend/src/modules/task-tree-feedback/`，后续扩展为 Runtime Feedback Surface。
- 工作区已有大量非本轮未提交改动；后续每次提交前需要用 `git status --short` 区分本轮新增与既有改动。

## 3. 回归清单

### 3.1 前端本地验证

- `cd web-admin/frontend && npm run build`
- 打开项目聊天页，确认消息列表、代码块复制按钮、代码预览按钮仍显示。
- 发送包含 `vue`、`html`、普通代码块的消息，确认只有可预览代码块显示预览按钮。
- 打开设置中心，确认路由、项目选择、模型设置、工具预算区域无明显异常。
- 打开任务树抽屉，确认任务树健康提示和节点状态仍可展示。

### 3.2 后端本地验证

- `cd web-admin/api && uv run pytest tests/test_project_chat_task_tree_routes.py tests/test_run_request_factory.py tests/test_tool_registry.py`
- 如拆分路由，额外运行 `tests/test_unit.py` 中项目聊天相关用例。
- 如修改 Runtime Core，额外运行 `tests/test_agent_orchestrator_runtime.py`。

### 3.3 发布前门禁

- 不允许新增业务逻辑继续进入 `ProjectChat.vue` 或 `projects.py` 的巨型主体。
- 每次抽取只移动一个职责域，先跑针对性验证，再继续下一步。
- 保持 API 路径兼容；若必须调整响应结构，需要同步前端适配和测试。
- 任务树、work session、requirement 状态必须继续复用 `project_id + chat_session_id + session_id`。

## 4. 风险记录

- 当前工作区有大量既有改动，不能用整体 diff 判断本轮范围。
- `ProjectChat.vue` 仍然巨大，短期内应避免大规模格式化，防止 diff 失控。
- 后端 `projects.py` 拆分前必须先补路由级回归，否则容易破坏素材/短片工作室等非聊天路径。
