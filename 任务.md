1. 手动切换模型：聊天时用文本模型，生成图片时切换图片模型。
2. 自动路由：系统判断用户意图；普通对话走文本模型，“生成图片”类请求才走图片模型。
3. 双模型配置：图片模型只负责生成，文本模型负责理解、优化提示词和普通对话。

• Codex 更接近方案 2：自动路由 / Agent 调用工具。

Codex 本身是文本与代码智能体，负责理解用户意图、规划任务，并调用工具或 Skill 完成工作，不是图片生成模型。(openai.com
(https://openai.com/index/introducing-codex/?utm_source=openai))

理想流程是：

用户消息
↓
Codex 判断意图
├─ 普通问答/代码任务 → Codex 直接回答
├─ 生成图片 → 调用图片生成模型
├─ 生成视频 → 调用视频生成模型
└─ 生成音频 → 调用音频生成模型

但当前刚修改的 ProjectChat 是方案 1：

- 选择 Codex/文本模型 → 正常对话和执行代码任务
- 选择图片模型 → 所有输入都作为图片提示词
- 选择视频模型 → 所有输入都作为视频提示词
- 不会由 Codex 自动决定是否调用图片模型

如果你希望“切换到 Codex 后既能聊天，又能说一句‘生成图片’直接出图”，下一步应该改成 Codex 主模型 + 图片/视频/音频工具，即方案 2，并可保留手动模型切换作为高级模式。
